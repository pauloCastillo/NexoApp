## ADDED Requirements

### Requirement: All timestamps use UTC
The system SHALL use UTC for all server-generated timestamps. Client-reported times (e.g., punch-in time) SHALL be stored as-is but the authoritative "recorded at" timestamp SHALL be UTC.

#### Scenario: Time control date uses UTC
- **WHEN** a new time control record is created
- **THEN** the `date` field SHALL default to `new Date()` (UTC) instead of `America/La_Paz`

#### Scenario: Work order date uses UTC
- **WHEN** a new work order is created
- **THEN** the `date` field SHALL default to `new Date()` (UTC) instead of `America/La_Paz`

#### Scenario: Location date uses UTC
- **WHEN** a new location record is created
- **THEN** the `date` field SHALL default to `new Date()` (UTC)

#### Scenario: All Date fields use UTC
- **WHEN** any model field of type `Date` is auto-generated by the server
- **THEN** it SHALL use UTC via `new Date()` or `DateTime.now().toUTC()`
